# 核心算法

## 集中式匹配

每个时间步，中央电脑执行：

```
1. 在位检测 — 无人机当前位置是否已在目标像素上
2. 空缺收集 — 收集所有未被填充的目标像素
3. 距离优先贪心分配 — 空位按到最近无人机的距离升序排列，依次匹配
4. 内层收敛 — 无人机向目标移动（运动学或动力学），未到达则重匹配
5. 收敛 → 推进到 t+1
```

## 去中心化匹配

无中央协调，无人机通过通信网络协商：

```
1. 在位检测 → 广播 CLAIM 消息
2. 各机收集已声明空缺，未分配者找最近未声明空缺
3. 广播新的 CLAIM → 冲突解决：同一空缺多人声明时距离近者胜
4. 渐进移动（每轮一小步）
5. 收敛判断 → 推进到 t+1
```

## 动力学

```
1. 期望速度 = 方向单位向量 × min(max_speed, dist/dt)
2. 所需加速度 = (v_desired - v_current) / dt
3. 阻力: a -= drag × v
4. 加速度 clamp 到 [-max_accel, max_accel]
5. 质量: a /= mass
6. 欧拉积分: v += a × dt, p += v × dt
```

## 碰撞规避

排斥力模型，在匹配决策之后微调速度：

```
F = strength × (1/d - 1/min_distance) / d²  （仅当 d < min_distance）
```

排斥力作为速度修正量直接叠加，不与匹配算法耦合。
